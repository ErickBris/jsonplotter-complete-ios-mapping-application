jsonPlotter Principles and Definitions:

Before we get into the nuts and bolts of the app it's important that we cover off the main principles of jsonPlotter. This will help you understand the full app functionality and also it's scope for deployment. Essentially the app is designed to take a json feed of location points (or entities) from a self hosted Wordpress install, parse and process these and then present the information on an iOS device. Wordpress posts are translated into entities which are displayed in both a map and a list view. An entity detail view is provided to display detailed information about each entity. An initial Chooser view is also provided to segment the entities allowing them to be grouped/categorized and then segmented by the end user.

This is an extremely versatile setup as it allows you to present various types of entities within the app. Usage examples include:

  • An coffee shop app that presents coffee shops (the entities) that are segmented (through the Chooser view) by city.
  • An app that creates different types of entities in one location, for example different types of restaurants in New York. The entity types to display are selected through the Chooser view.

If you open and run the included app you will see that the app and example data utilized presents coffee shops in various cities in the US. In this case the coffee shops are the entities (Wordpress posts) and the following views apply:

The Chooser View. This view in which you choose how you would like to segment your entities.

The Map View. This is where the map pins of the entities (segmented by the Chooser View) are presented. User location is also shown allowing users to pinpoint their location and view map pin proximity.

The List View, This is a list view of the where a list of entities (segmented by the Chooser View) are presented. This view can help users find specific entities they are looking for and includes a handy search function to aid with this.

The Detail View shows further details for the entity. This view displays images, a zoomed map location, entity detail and also provides a Twitter share button so users can share entity listings socially.

All data is fed into the app through a Wordpress Post json feed of allowing you to easily update the data presented in the app whenever required. App data is initially called on app load and stored in an internal app database. Data is refreshed when the app is place in the background memory ensuring all listing s are kept up to date. The database can also be refreshed on demand by the user at any time by using the refresh button (top right of app view.)A number of checks are made also by the app to ensure a web connection is present, if not the user is notified.

The jsonPlotter example app included is designed to display coffee shop entities across 5 American cities. Users can choose which city they would like to view, corresponding entities (i.e. the coffee shops in the city chosen) are displayed. To guide you through the inner working of the app example provide and reveal the scope for customization we will now guide you through how the included app (and corresponding json feed) was setup. The whole process can be completed in under 1 hour, in fact, if all things are properly setup, this could take just 15 minutes!

Application Setup:

Application setup can be facilitated by following the 4 steps as outlined below:

1 - Install Wordpress

You will need a working Wordpress installation to feed data into your application. Simply install Wordpress onto your server as per usual - detailed information + instructions regarding the installation and setup of WordPress please follow the WordPress Codex Installation Instructions found here:

http://codex.wordpress.org/Installing_WordPress

 

2 - Json api Plugin Install

Install the following plugin: JSON API (A RESTful API for WordPress) by Dan Phiffer (Dan this is awesome by the way!) Using this plugin you can create a a json feed of your Wordpress posts. This is the feed that will be passed into and utilized by the jsonPlotter app.

Once this plugin is installed you can view/test the feed by adding "api/get_recent_posts" to your base url, i.e. use following URL construction:

http://YOUR-SITE-ADDRESS/api/get_recent_posts

Our example of the feed being passed into the included app uses http://amplifiii.com/jsonPlotter/api/get_recent_posts/

 

3 - Setup your json feed:

Post data is fed through the feed into the app using both standard post information (title, body etc) and by using additional custom post types to satisfy entity data/information requirements. In our example/the initial application build included we have utilized the following standard and custom post information to map to our entity data requirements:

  • Entity Title = Standard Wordpress Post Title
  • Entity Description = Standard Wordpress Post Description/Body
  • Entity Image = Standard Wordpress Post Featured Image
  • Entity Address = An "Address" Custom Post Type
  • Entity Latitude = A "Latitude" Custom Post Type (essential for plotting data point on map.)
  • Entity Longditude = A "Longditude" Custom Post Type (essential for plotting data point on map.)

The latitude and longitude data custom post types are essential for plotting map points, without these the map view will not be able to plot the entities. You can convert an address to latitude and longditude data using one of the awesome converters that can be found online.

Aside from Latitude and Longditude information all other data points included are used to present information within the map and can be edited as desired.

Once we have setup our posts as required and inserted app data the relevant information can be pulled through the feed by adjusting the json URL call as follows:

http://amplifiii.com/jsonPlotter/api/get_recent_posts/?count=2000&custom_fields=Address,City,Phone%20Number,Website%20Address,Latitude,Longitude,thumbnail=full,Postend&page=%ld

If you copy and paste this into a browser you can see how the resulting data is presented. Further information on how to construct json URL calls can be found on the json plugin web page.

Points to Note:

  • The chooser elements within the app are initially setup as "City" custom post types. These can be re-named as you please however you will need to update the custom post type name and the name of the elements established within the app.
  • Location data should be stored as Longitude and Latitude custom post types, without these the app will not be able to plot any pins on the map (as it wont know where they are meant to be plotted).
  • The Pin annotation detail view is initially mapped to an "Address" custom post type however this can be changed to satisfy any data type requirements you may have, as with the City post type.
